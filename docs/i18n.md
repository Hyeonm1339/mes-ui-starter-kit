# 다국어(i18n) 등록 가이드

번역 키를 추가하고 컴포넌트에서 사용하는 방법을 설명합니다.

---

## 번역 파일 위치

| 언어 | 파일 |
|---|---|
| 한국어 | `src/locales/ko/common.json` |
| 영어 | `src/locales/en/common.json` |

기본 언어는 한국어이며, 헤더의 KO/EN 버튼으로 전환됩니다.

---

## 네임스페이스 구조

`common.json`은 앱 전역에서 공통으로 쓰이는 키를 관리합니다.

```jsonc
{
  "nav":    { ... },   // 메뉴 레이블
  "auth":   { ... },   // 로그인/인증
  "common": { ... },   // 범용 메시지
  "layout": { ... }    // 레이아웃 안내 문구
}
```

피처별로 번역이 많다면 별도 네임스페이스 파일로 분리할 수 있습니다.  
예시: `src/locales/ko/workOrder.json` → `useTranslation('workOrder')`

---

## 키 추가 방법

### 1. 네비게이션 레이블

새 메뉴를 추가할 때마다 `nav` 객체에 키를 추가합니다.

```json
// src/locales/ko/common.json
{
  "nav": {
    "production":  "생산관리",
    "workOrder":   "작업지시",
    "workOrderList": "작업지시 목록"
  }
}
```

```json
// src/locales/en/common.json
{
  "nav": {
    "production":  "Production",
    "workOrder":   "Work Order",
    "workOrderList": "Work Order List"
  }
}
```

### 2. 피처 전용 번역

번역 키가 많은 피처는 별도 파일로 분리합니다.

```jsonc
// src/locales/ko/workOrder.json
{
  "title": "작업지시 목록",
  "search": {
    "woNo":     "작업지시번호",
    "itemName": "품목명"
  },
  "status": {
    "pending":    "대기",
    "inProgress": "진행중",
    "done":       "완료"
  }
}
```

새 파일을 추가하면 `src/lib/i18n.tsx`의 리소스에도 등록해야 합니다.

```ts
// src/lib/i18n.tsx
import koWorkOrder from '@/locales/ko/workOrder.json'
import enWorkOrder from '@/locales/en/workOrder.json'

resources: {
  ko: { common: koCommon, workOrder: koWorkOrder },
  en: { common: enCommon, workOrder: enWorkOrder },
}
```

## 컴포넌트에서 사용

### 기본 사용

```tsx
import { useTranslation } from 'react-i18next'

const MyComponent = () => {
  const { t } = useTranslation()   // 기본 네임스페이스: common

  return <h1>{t('nav.workOrder')}</h1>
}
```

### 피처 전용 네임스페이스

```tsx
const { t } = useTranslation('workOrder')

return <h1>{t('title')}</h1>   // workOrder.title
```

### 변수 삽입

```tsx
// 번역 파일: "total": "총 {{count}}건"
t('total', { count: 42 })   // → "총 42건"
```

### 기본값 (키가 없을 때)

```tsx
t('some.key', { defaultValue: '기본 텍스트' })
```

---

## 미사용 키 관리 규칙

- 기능이 제거될 때 해당 번역 키도 **함께 삭제**합니다.
- 현재 `common.json`에는 실제로 사용 중인 키만 포함되어 있습니다.
- 향후 기능 추가 시 해당 시점에 필요한 키를 추가합니다.
